# 张小珺商业访谈提问法

> "如果我的访谈能陪你走一段孤独的未知的路，也许有一天可以离目的地更近一点，我就很温暖。"

一套可操作的深度访谈提问框架，基于对《张小珺商业访谈录》全部节目的系统观察和逆向工程提炼而成。

## 核心理念

五个底层假设：

1. **连续性 > 一次性**：好的访谈不是偶遇，是长期跟踪的一个截面
2. **场景 > 提纲**：用具体的场景把听众和嘉宾一起拉进去
3. **结构 > 碎片**：2-7小时的长对话有清晰的结构推进
4. **信任 > 犀利**：能触及敏感话题的前提是前面建立了足够的专业信任
5. **画像 > 信息**：先勾勒一幅人物画像，再开始对话

## 11个提问模型

| # | 模型 | 一句话 | 适用场景 |
|---|------|--------|---------|
| 1 | 场景叙事开场法 | 时间+地点+氛围，把听众拉进去 | 开场 |
| 2 | 趋势锚定法 | 锚定可验证的趋势信号 | 行业话题 |
| 3 | 原话回钩法 | 用对方说过的话做钩子 | 返场嘉宾 |
| 4 | 结构追问法 | 结构→机制→数据→反例 | 对方给出判断后 |
| 5 | 数据锚定法 | 用具体数据点做锚 | 需要量化讨论时 |
| 6 | 信任递进法 | 行业→业务→个人，按节奏推进 | 整体编排 |
| 7 | 嘉宾画像勾勒法 | 视觉细节+身份标签+一句话定调 | 介绍嘉宾时 |
| 8 | 新概念定义法 | 先定义再讨论 | 行业新概念出现时 |
| 9 | 马拉松对话法 | 7小时=从职业模式到真人模式 | 超长访谈 |
| 10 | 返场关系构建法 | "第N次访谈"的长期叙事 | 老嘉宾 |
| 11 | 三层递进提问法 | 共识→事实→矛盾，防止问题太跳 | 所有核心问题 |

## 提问设计原则

### 原则0：当下锚定

所有问题必须紧扣"当下"——这个"当下"指的是你提问的那个时间点。

- 论证赛道要用此刻最有说服力的产品
- 不要用过时的参照系
- 不要翻几年前的旧账论证现在的选择
- 检验方法：把时间锚点换成"去年"，如果意思差不多，说明不够"当下"

### 原则1：问题要短，留白要足

✅ "ChatGPT会成为一个新的超级入口吗？"
❌ "现在很多人都在讨论ChatGPT会不会成为一个超级入口，你觉得它会不会对Meta构成威胁，以及它的商业模式最终会怎样？"

### 原则2：问题要有锚点

不要问"你怎么看AI"，要问"Sam Altman今年几乎不提AGI了，你觉得这意味着什么"。

### 原则3：问题要是开放式的

用"你觉得"而非"你同意吗"。

## 事实校准纪律

提问之前，以下事实必须逐一核实：

| # | 校准项 | 说明 |
|---|--------|------|
| 1 | 人物身份 | 不能把A公司的创始人当成B公司的 |
| 2 | 产品关系 | A产品用了谁的模型、和谁合作，必须准确 |
| 3 | 融资状态 | 对方是缺钱还是不缺钱，要搞清楚 |
| 4 | 时间敏感性 | 引用的事件/产品必须是当下的 |
| 5 | 行业共识 | "共识"必须真的是共识 |
| 6 | 论证选择 | 论证赛道要用此刻最有说服力的产品 |

## 反模式黑名单

| # | 反模式 | 为什么不做 |
|---|--------|----------|
| 1 | 开场就问私人问题 | 信任未建立，对方会启动防御模式 |
| 2 | 问题比答案还长 | 暴露不安全感，限制嘉宾表达空间 |
| 3 | "你同意吗？"式提问 | 把对话变成二元选择题 |
| 4 | 不做功课就来采访 | "几十页提纲"不是夸张，是底线 |
| 5 | 一次性消费嘉宾 | 持续追踪才能建立深度和信任 |
| 6 | 回避矛盾和争议 | 但要用"去年那场意外"的方式，不是审判式 |
| 7 | 追求金句而牺牲深度 | 2-7小时的价值在于思考过程 |
| 8 | 用"大家都说"代替具体数据 | "10.3亿美元Seed轮"比"很多钱"有力量 |
| 9 | 假设听众知道所有概念 | Neo Labs、Post-train——先定义再讨论 |
| 10 | 干巴巴的开场 | "XX你好"不如"2026年春节，纽约刚下过一场暴雪" |
| 11 | 事实错误 | 人物搞混、产品关系乱——一个错误毁掉整场信任 |
| 12 | 用过时参照系 | 说明你没在跟踪当下 |
| 13 | 问题转折太跳 | 缺事实铺垫，对方一句套话就打发了 |
| 14 | 论证选错产品 | 要用此刻最有说服力的玩家，不是边缘玩家 |
| 15 | 翻旧账论证现在 | 用几年前的决策论证现在的选择，逻辑不成立 |

## 快速启动清单

如果你明天就要做一场深度访谈：

- [ ] 找到一个"时间锚点"，写一个场景式开场
- [ ] 把嘉宾所有公开言论过一遍，标出3-5个"原话回钩"点
- [ ] 如果嘉宾之前来过，设计"第N次访谈"的叙事
- [ ] 找到行业最近1-2个关键趋势信号——必须是"当下"的
- [ ] 设计4个板块的问题树（行业→技术/业务→个人→感悟）
- [ ] 每个板块有1个"杀手级问题"——对方大概率没被问过的
- [ ] 问题全部改短，删掉所有背景铺垫
- [ ] 核心问题用"三层递进"结构：共识→事实→矛盾
- [ ] 逐条过「事实校准纪律」表
- [ ] 准备好"如果对方回避怎么办"的备用追问路径
- [ ] 准备1-2个"五个字"式问题
- [ ] 勾勒一幅嘉宾画像（视觉细节+身份标签+一句话定调）
