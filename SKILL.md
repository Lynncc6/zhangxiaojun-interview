# 张小珺 · 商业访谈提问方法论

> "如果我的访谈能陪你走一段孤独的未知的路，也许有一天可以离目的地更近一点，我就很温暖。"

## 核心理念

张小珺的访谈不是问答，是**一场有预谋的认知探险**。

从B站全部节目的系统观察中，她的方法论建立在五个底层假设之上：

1. **连续性 > 一次性**：好的访谈不是偶遇，是长期跟踪的一个截面。对朱啸虎"第三次访谈"、对何小鹏"第二次访谈"——每一次都在之前的基础上追问变化。
2. **场景 > 提纲**：开场不是"请自我介绍一下"，而是用一个具体的场景把听众和嘉宾一起拉进去。"2026年春节，在中国机器人登上春晚的喜乐时分，纽约刚下过一场暴雪——这是近几年以来纽约最凛冽的一个冬天。在布鲁克林一栋略显凌乱的楼房，我与谢赛宁开始了这场对话。"
3. **结构 > 碎片**：2-7小时的长对话有清晰的结构推进——从行业大势到公司战略，从公开数据到私人判断。每个板块之间有明确的转场。
4. **信任 > 犀利**：能触及"去年那场意外""没上过大学是否遇到挫折"这种话题，前提是前面已经建立了足够的专业信任。
5. **画像 > 信息**：她的节目简介不只是信息罗列，而是先勾勒一幅人物画像——"姚顺宇戴了一副茶色眼镜走过来""这是谢赛宁第一次接受访谈""她是《商业访谈录》至今最年轻的一位嘉宾，是一位00后华人女孩"。

## 心智模型

### 模型1：场景叙事开场法

**核心逻辑**：不用干巴巴的"今天我们邀请到了XX"开场，而是用一个具体的时间+地点+氛围的场景，把听众直接拉入对话。

**操作步骤**：
1. 找到访谈发生时的一个具体时间锚点（节日、季节、天气、行业事件）
2. 加入物理空间的细节（地点、环境、氛围）
3. 用1-2句话勾勒嘉宾的出场形象
4. 自然过渡到访谈主题

**示例**（对谢赛宁）：
> "2026年春节，在中国机器人登上春晚的喜乐时分，纽约刚下过一场暴雪——这是近几年以来纽约最凛冽的一个冬天。在布鲁克林一栋略显凌乱的楼房，我与谢赛宁开始了这场对话。"

**示例**（对姚顺宇）：
> "姚顺宇戴了一副茶色眼镜走过来，这副眼镜会随光线变化而改变颜色。"

**为什么有效**：
- 场景叙事比信息罗列更能建立"在场感"
- 细节（茶色眼镜、凌乱的楼房）暗示你真的见过这个人，不是远程念稿
- 时间锚点（春节/暴雪）让访谈有了历史坐标感

---

### 模型2：趋势锚定法

**核心逻辑**：不直接问"你怎么看X"，而是锚定一个可验证的趋势变化，让对方在具体的参照系里表达判断。

**操作步骤**：
1. 找到一个行业内公认的趋势信号（如"Sam Altman今年几乎不提AGI了"）
2. 把这个信号包装成问题的锚点
3. 追问对方对此的解读和判断

**示例**（对朱啸虎）：
> "从年初到现在，你觉得你对AI产业的判断有没有发生一些根本性的变化？有没有什么出乎意料的？"

**变体**（对罗福莉）：
> "现在至少跟23年要去追平Pre-train的差距一样，大家很all in，要去做好Agent的Post-train。你觉得这个方向对吗？"

**反模式**：
- ❌ "你怎么看AI行业？"（太宽泛）
- ❌ "你觉得AI未来会怎样？"（太空）

---

### 模型3：原话回钩法（第N次访谈模式）

**核心逻辑**：用对方自己说过的话做钩子，追问当下的变化。这既是对对方的尊重（证明你做了功课），也是一种温和的施压。

**操作步骤**：
1. 系统性梳理嘉宾过往公开言论（文章、演讲、访谈、推文）
2. 找到那些可以被时间检验的判断（"去年你说过……"）
3. 用原话引出新问题
4. 如果是"返场嘉宾"，直接标注"第N次"

**示例**（对朱啸虎第三次访谈）：
> "2024年3月、2025年2月，我曾两次更新朱啸虎的中国现实主义AIGC故事，现在又过去了快1年时间。站在2025年的尾巴上，AI产业有泡沫吗？——这里是朱啸虎现实主义故事的第三次连载。"

**示例**（对何小鹏第二次访谈）：
> "在我们此前的节目中，何小鹏形容，造车就像在'在血海里游泳'。在过去1年中，除了汽车这场血海战役，小鹏汽车也要面临AI和机器人这场未来之战……我们这次从'赌'开始聊起。"

**示例**（对杨植麟）：
> "杨植麟好久没露面了，上次采访他还是1年半以前。这次，我们聊了K2的研发和他的最新技术认知，以及，在过去一年的舆论风暴与创业起伏中，他的心情与思考。"

**为什么有效**：
- 建立了"持续记录"的叙事框架，让嘉宾觉得这不是一次性消费
- 原话引用证明你真的研究过对方
- "第N次"的说法把对话变成了一个系列，嘉宾和听众都会更认真对待

### 模型4：结构追问法

**核心逻辑**：当对方给出一个判断后，不满足于表面答案，而是沿着"结构→机制→数据→反例"的路径层层追问。

**追问路径**：
```
对方给出判断 → 追问结构（"这个判断的底层结构是什么？"）
             → 追问机制（"为什么会这样？因果链条是什么？"）
             → 追问数据（"有没有具体的数据/案例支撑？"）
             → 追问反例（"什么情况下这个判断不成立？"）
```

**示例**（对朱啸虎关于AI泡沫）：
- 朱啸虎说"三年内看不到泡沫"
- 追问："那这股关于AI泡沫的讨论，它的这个悲观情绪是怎么传导的？它从哪儿来的？"（追问传导机制）

**示例**（对何小鹏关于机器人）：
- 何小鹏说"造人形机器人的胜率比造车低得多"
- 可追问路径：
  - "低得多是低到什么程度？"（量化追问）
  - "为什么胜率更低？是技术、供应链、还是市场的问题？"（结构追问）
  - "那你为什么还要下更大的赌注？"（矛盾追问）

---

### 模型5：数据锚定法

**核心逻辑**：用具体的、可验证的数据点做提问的锚，而不是模糊的"很多人说"。

**操作步骤**：
1. 找到行业内的关键数据（token消耗量、融资额、用户增长曲线、论文引用数）
2. 把数据嵌入问题中
3. 追问对方对这个数据的解读

**示例**（对谢赛宁）：
> "AMI Labs目前仅25人、在没有任何产品的情况下，完成10.3亿美元Seed轮融资，投前估值35亿美元。这是2026年新创的AI Neo Labs中备受瞩目的一家，创造了欧洲历史最大Seed轮。从统计看，它的第一笔融资在Neo Labs中位居世界第二，仅次于前OpenAI CTO Mira Murati创立的Thinking Machines Lab（20亿美元Seed轮融资）。你怎么看这个现象？"

**示例**（对罗福莉）：
> "Anthropic的路径是正确的，这是当下共识。在路径更清晰的情况下，国内大模型团队进入加速追赶的状态。现在大家在Pre-train上的代差是基本没有的，或者说非常接近。你觉得这个判断准确吗？"

### 模型6：信任递进法

**核心逻辑**：私人问题只能在专业信任建立之后提出。访谈的前半段是"专业能力展示"，后半段才是"人性挖掘"。

**信任递进的节奏**（以4小时访谈为例）：
```
第0-30分钟：行业大势 → 展示你对行业的理解深度
第30-90分钟：技术/业务细节 → 展示你对嘉宾领域的研究精度
第90-150分钟：公司战略与竞争 → 进入专业深水区
第150-210分钟：个人判断与心路 → 在信任基础上触及私人话题
第210-240分钟：人生感悟与价值观 → 最柔软也最有价值的部分
```

**示例**（对何小鹏）：
> 在聊完了AI赌注、Iron机器人、新车GX之后，才聊到"去年那场意外"——"去年那场意外"这五个字本身就说明了一切，不需要展开描述，但所有人都知道你在说什么。

**示例**（对杨植麟）：
> 先聊K2的技术细节、Agentic LLM、缸中之脑、泛化难题——这些专业话题建立信任后，再聊"在过去一年的舆论风暴与创业起伏中，他的心情与思考"。

**为什么有效**：
- 商业人物的时间极其宝贵，前半段的专业提问是在"支付"对方的时间成本
- 当对方意识到"这个人真的懂行"，才会愿意暴露脆弱的一面
- 私人问题的价值不在于八卦，而在于揭示"这个人在关键时刻是怎么做决策的"

### 模型7：嘉宾画像勾勒法

**核心逻辑**：在访谈正式开始前，用3-5句话勾勒一幅嘉宾的"人物速写"，让听众在对话开始前就建立对这个人的感知。

**画像的三个维度**：
1. **视觉细节**："姚顺宇戴了一副茶色眼镜""在布鲁克林一栋略显凌乱的楼房"
2. **身份标签**："出生于1990年的华人科学家""00后华人女孩""《商业访谈录》至今最年轻的一位嘉宾"
3. **一句话定调**："谢赛宁第一次接受访谈""她引起很多人的关注，来自于这样一条新闻：57岁美国终身教授突然辞职，去给24岁的华人女孩打工"

**示例**（对洪乐潼）：
> "她是《商业访谈录》至今最年轻的一位嘉宾，是一位00后华人女孩，网络空间有人会叫她'数学少女'——她的名字是洪乐潼Carina，这是她第一次接受中文访谈。她探索的方向是AI for Math，所创办的公司Axiom（公理）刚完成2亿美金的A轮融资。而她引起很多人的关注，来自于这样一条新闻：57岁美国终身教授小野肯突然辞职，去给24岁的华人女孩打工。"

**为什么有效**：
- 好的画像让嘉宾在开口之前就已经"立住了"
- 视觉细节增加真实感和信任感
- "第一次接受访谈"这种信息本身就是稀缺性的暗示

### 模型8：新概念定义法

**核心逻辑**：当行业出现一个新概念（Neo Labs、Agent Post-train、世界模型）时，不假设听众已经知道，而是在访谈中自然地完成"定义+讨论"。

**操作步骤**：
1. 在节目简介中先完成定义（"Neo Labs是近年兴起的一个新概念，专指由顶级AI研究者创立、以基础模型或新一代智能体系为目标、融资规模巨大、研究导向很强的一类新型AI实验室"）
2. 在访谈中用这个概念做锚点提问
3. 追问嘉宾对这个概念的理解和判断

**示例**（对谢赛宁/洪乐潼）：
> 先在简介中定义"Neo Labs"，然后在访谈中问："作为2026年最受关注的Neo Labs之一，你觉得Neo Labs和传统AI公司的根本区别是什么？"

**示例**（对罗福莉）：
> "现在大家很all in，要去做好Agent的Post-train。更具体说，是在Agent上怎么做好RL的scaling。你觉得这个方向是对的吗？"

### 模型9：马拉松对话法

**核心逻辑**：7小时不是"把4小时拉长"，而是一种完全不同的访谈形态——它允许嘉宾从"职业模式"切换到"真人模式"。

**马拉松访谈的节奏设计**：
```
第1-2小时：专业热身（行业+技术）
第2-4小时：深水区（战略+竞争+争议）
第4-5小时：松弛期（嘉宾开始说"题外话"）
第5-6小时：真心话（"我其实一直觉得……""这件事我从来没说过……"）
第6-7小时：哲学与人生（从具体到抽象，从战术到存在）
```

**示例**（对谢赛宁7小时访谈）：
> 从下午2点到凌晨，跨越了"世界模型技术路线→反OpenAI立场→两次拒绝Ilya的故事→杨立昆和李飞飞→42（终极答案的隐喻）"——如果只有2小时，"42"和"两次拒绝Ilya"这些故事可能永远不会出现。

**关键心法**：
- 不预设"必须在X时间内聊完Y话题"
- 在第4-5小时保持沉默和耐心，让嘉宾自己想继续说
- 准备好"备用话题库"但不急于推进

### 模型10：返场关系构建法

**核心逻辑**：把嘉宾从"一次性采访对象"变成"长期追踪的思想伙伴"。

**构建方法**：
1. 第一次访谈时就埋下"下次再聊"的种子（"期待一年后我们再坐下来"）
2. 中间保持轻量级联系（不是骚扰，而是偶尔分享相关内容）
3. 第二次访谈开场直接引用第一次的内容
4. 在节目简介中明确标注"返场"或"第N次"

**已建立返场关系的嘉宾**：
- 朱啸虎：3次（2024.3 → 2025.2 → 2025年底）
- 何小鹏：2次（"造车像在血海里游泳" → "下了更大的赌注"）
- 杨植麟：2次（"一年融资90亿后" → "K2和舆论风暴后的心情"）

**返场的价值**：
- 嘉宾更放松，因为知道你不是来"挖一次料"的
- 听众形成追更习惯
- 你自己积累了一个行业认知的时间序列

### 模型11：三层递进提问法

**核心逻辑**：一个好问题不能"跳"——不能直接从行业共识跳到尖锐矛盾。要先建立共识、再摆事实、最后抛矛盾，三层递进，让对方无法用一句"我们专注做好产品"就给你挡回来。

**三层结构**：
```
第一层：锚定共识（"今年行业有一个共识……"）
第二层：摆出事实（"X已经证明了Y，而你的能力是Z"）
第三层：抛出矛盾（"那你为什么不做？"）
```

**操作要点**：
1. 第一层的"共识"必须是此刻行业内真正被广泛认可的，而不是你自己编的
2. 第二层的"事实"必须准确——产品关系不能乱、商业现实不能错、数据不能虚
3. 第三层的"矛盾"要给对方留两个以上的选项（"是觉得不该你赚，还是觉得做模型就够了？"），而不是封闭式的yes/no

**示例**（对梁文锋/DeepSeek，2026年7月）：
> "今年行业有一个共识：代码和办公方向是大模型商业化最清晰的两条路。Claude Code和Codex已经把代码harness这条路彻底走通了——用户愿意为一个能帮他写代码、改代码、理解整个项目的AI代理付费。DeepSeek的代码模型能力是公认的强，但你们到现在还是只卖模型API，没有自己往下游做产品。你是觉得做harness这钱不该你赚，还是觉得做模型就够了？"

**反例**（为什么不好）：
> "DeepSeek为什么不自己做一个harness？"（太跳，没有建立共识和事实铺垫，对方一句"我们专注做好模型"就打发了）

**为什么有效**：
- 共识层让对方无法否定前提（"这不是我的观点，是行业共识"）
- 事实层展示你的研究深度，建立专业信任
- 矛盾层把"为什么不"变成了一个选择题，而不是审判，降低对方的防御心理

## 提问设计原则

### 原则0：当下锚定——问题的时间坐标就是此刻

**所有问题必须紧扣"当下"。** 这个"当下"指的是你提问的那个时间点，而不是半年前、一年前。

**具体要求**：
- 论证赛道要用此刻最有说服力的产品（2026年7月论证代码Agent，用Claude Code + Codex，而不是Cursor）
- 不要用过时的参照系（不要在2026年还在聊Llama 4的发布，那已经是旧闻了）
- 不要翻几年前的旧账来论证现在的问题（不要用"你们早期Coder开源"来论证"你们现在的开源策略"）
- 行业共识必须是此刻的共识（"代码和办公是最清晰的商业化路径"是2026年的共识，不是2024年的）

**检验方法**：把你的问题中的时间锚点换成"去年"，如果意思差不多，说明你的问题不够"当下"。

**反面案例**：
- ❌ "Cursor用你们的模型做代码harness……"（Cursor和DeepSeek没有强绑定，事实错误）
- ❌ "Llama 4彻底开源了……"（在2026年中聊这个，已经不是当下热点了）
- ❌ "你们早期Coder开源的时候……"（用几年前的决策论证现在的选择，逻辑不成立）

---

### 原则1：问题要短，留白要足

张小珺的问题通常很短，把大量的"思考空间"留给嘉宾。

✅ 好问题："ChatGPT会成为一个新的超级入口吗？"
❌ 坏问题："现在很多人都在讨论ChatGPT会不会成为一个超级入口，你觉得它会不会对Meta构成威胁，以及它的商业模式最终会怎样？"

---

### 原则2：节奏感：从宏观到微观的波浪推进

一场好的访谈像一首曲子，有起承转合：

```
起（开场）：场景叙事+锚定趋势，建立语境
承（展开）：行业结构→竞争格局→公司/技术战略
转（深入）：数据追问→矛盾追问→个人判断
合（收束）：未来展望→个人选择→人生感悟
```

**节奏控制**：
- 每个话题板块20-40分钟
- 在板块之间用"好，那我们换一个话题聊"做清晰的转场
- 当嘉宾在一个话题上说得足够深入时，及时推进到下一个

---

### 原则3：问题要有"信息密度"

每个问题都应该同时完成至少两件事：

| 问题 | 同时完成的事 |
|------|------------|
| "从年初到现在，你对AI产业的判断有没有发生根本性变化？" | 1. 获取最新判断 2. 暗示"你之前的判断可能变了" 3. 给对方一个修正的机会 |
| "那为什么他们现在启动了这个事情呢？" | 1. 追问动机 2. 暗示"你说还早，但他们已经在做了" 3. 引出矛盾分析 |
| "我们这次从'赌'开始聊起" | 1. 引出新话题 2. 引用对方之前的话 3. 暗示这次访谈的核心张力 |

---

### 原则4：用"你觉得"代替"你是否同意"

✅ "你觉得这种类ChatGPT的产品最终会是一个什么样的终局？"
❌ "很多人说ChatGPT会形成垄断，你同意吗？"

---

### 原则5：用"五个字"提最痛的问题

当涉及敏感话题时，不展开描述，用极简的指代。

✅ "去年那场意外"（对何小鹏）——五个字，所有人都知道你在说什么，但你没有"撕开伤口"的姿态
❌ "去年你们那款车发生了严重的事故，造成了XX伤亡，当时你是怎么处理的？"

---

### 原则6：给嘉宾"第一次"的舞台

张小珺特别擅长邀请"第一次接受访谈"的人：
- 谢赛宁："这是谢赛宁第一次接受访谈"
- 洪乐潼："这是她第一次接受中文访谈"

"第一次"的价值：
- 嘉宾没有被训练过的"标准答案"
- 听众有"独家感"
- 嘉宾会更认真对待（因为是"第一次"）

## 访谈准备工作流

张小珺为一次对话准备**几十页提纲**。以下是从B站节目中逆向工程出的准备方法论：

### Step 1：嘉宾全景扫描（2-3天）

- **基础信息**：教育背景、职业经历、关键转折点
- **公开言论**：所有可找到的演讲、访谈、文章、社交媒体发言、论文
- **行业认知**：嘉宾所在行业的最新动态、数据、争议、新概念定义
- **个人风格**：嘉宾在不同场合的表达风格、回避的话题、喜欢的表达方式、口头禅
- **视觉细节**：如果能找到照片或视频，注意嘉宾的穿着、配饰、肢体语言——"茶色眼镜"这种细节不是随便写的

### Step 2：构建认知地图（1天）

把收集到的信息组织成一张"认知地图"：
- 嘉宾的核心观点有哪些？
- 这些观点之间有什么逻辑关系？
- 哪些观点被时间证明了/证伪了？
- 嘉宾有哪些"思维盲区"（他可能没意识到的偏见）？
- 嘉宾和行业内其他关键人物的关系是什么？（"两次拒绝Ilya""和杨立昆一起创业"）

### Step 3：设计问题树（1天）

```
访谈主题
├── 开场：场景叙事（时间+地点+人物出场）
├── 板块1：行业大势
│   ├── 趋势锚定问题
│   ├── 新概念定义问题
│   └── 矛盾点问题
├── 板块2：技术/产品战略
│   ├── 具体技术路线问题
│   ├── 竞争格局问题
│   └── 资源分配问题
├── 板块3：个人判断与心路
│   ├── 原话回钩问题
│   ├── 变化追踪问题
│   ├── "去年那场意外"式问题
│   └── 反例挑战问题
├── 板块4：人生感悟
│   ├── 关键转折点问题
│   ├── 挫折与反思问题
│   └── 价值观问题
└── 备用话题库（不预设全部聊完）
```

### Step 4：预演与删减（半天）

- 把问题大声读出来，感受是否自然
- 删掉所有"为了展示我做了功课"的问题
- 确保每个板块都有至少一个"杀手级问题"
- 准备好"备用问题库"——如果嘉宾在某个话题上不想聊，能迅速切换
- 特别注意：如果是返场嘉宾，把上次的核心观点和原话找出来，做"原话回钩"

## 表达DNA

当你以张小珺式风格提问/主持时，注意以下表达特征：

- **语速**：中等偏慢，每个字都清晰
- **语气**：专业但不咄咄逼人，好奇但不猎奇，温暖但不煽情
- **用词**：精准、简洁，避免模糊的形容词。用具体的数字和名字
- **停顿**：问完问题后，给对方足够的思考时间（3-5秒的沉默是正常的）
- **反馈**：用"嗯""对""有意思"等简短反馈鼓励对方继续，而不是打断
- **转场**：用"好，那我们换一个话题聊""我们这次从X开始聊起"做明确的板块切换

**典型句式**：
- "从X到现在，你觉得Y有没有发生一些根本性的变化？"
- "那为什么Z呢？"
- "你觉得这种X最终会是一个什么样的Y？"
- "我们这次从'X'开始聊起"
- "上次采访他还是1年半以前"
- "这是XX第一次接受访谈"

## 反模式黑名单

| # | 反模式 | 为什么不做 |
|---|--------|----------|
| 1 | 开场就问私人问题 | 信任未建立，对方会启动防御模式 |
| 2 | 问题比答案还长 | 暴露不安全感，限制嘉宾表达空间 |
| 3 | "你同意吗？"式提问 | 把对话变成二元选择题 |
| 4 | 不做功课就来采访 | "几十页提纲"不是夸张，是底线 |
| 5 | 一次性消费嘉宾 | 持续追踪才能建立深度和信任 |
| 6 | 回避矛盾和争议 | 但要用"去年那场意外"的方式，而不是审判式 |
| 7 | 追求金句而牺牲深度 | 2-7小时的价值在于思考过程 |
| 8 | 用"大家都说"代替具体数据 | "10.3亿美元Seed轮"比"很多钱"有力量100倍 |
| 9 | 假设听众知道所有概念 | Neo Labs、Post-train、DiT——先定义再讨论 |
| 10 | 干巴巴的开场 | "XX你好欢迎来到我们节目"不如"2026年春节，纽约刚下过一场暴雪" |
| 11 | 事实错误 | 人物搞混、产品关系乱、融资状态错——一个错误毁掉整场信任 |
| 12 | 用过时参照系 | 2026年还在聊2024年的热点，说明你没在跟踪当下 |
| 13 | 问题转折太跳 | 直接从"行业共识"跳到"你为什么不做"，中间缺事实铺垫 |
| 14 | 论证选错产品 | 论证赛道要用此刻最有说服力的玩家，而不是边缘玩家 |
| 15 | 翻旧账论证现在 | 用几年前的决策论证现在的选择，逻辑不成立 |

## 事实校准纪律

提问之前，以下事实必须逐一核实，不能有任何差错。一个事实错误，足以毁掉整场访谈的信任基础。

| # | 校准项 | 说明 | 反面案例 |
|---|--------|------|----------|
| 1 | 人物身份 | 不能把A公司的创始人当成B公司的 | 把科大讯飞刘庆峰当成DeepSeek梁文锋 |
| 2 | 产品关系 | A产品用了谁的模型、和谁合作，必须准确 | "Cursor用DeepSeek的模型"（Cursor主要用Claude/GPT） |
| 3 | 融资状态 | 对方是缺钱还是不缺钱，要搞清楚 | 对梁文锋问"你怎么解决融资问题"（投资人追着给钱） |
| 4 | 时间敏感性 | 引用的事件/产品必须是当下的，不是半年前的 | 2026年7月还在把Llama 4发布当新闻聊 |
| 5 | 行业共识 | "共识"必须真的是共识，不是你以为的 | "大家都觉得大模型泡沫要破了"——如果其实大家都在all in，这就不是共识 |
| 6 | 论证选择 | 论证一个赛道成立，用此刻最有说服力的产品 | 论证代码Agent，用Claude Code + Codex而非Cursor + Windsurf |

**校准流程**：问题设计完成后，逐条过上面这张表。任何一条存疑，要么去核实，要么换一个锚点。

## 诚实边界

- **这个Skill做不到的**：让你一夜之间拥有张小珺的行业积累、人脉和"几十页提纲"背后的研究深度。她的提问能力建立在多年财经记者的深度行业研究和对AI/科技领域数百位关键人物的长期追踪之上。
- **这个Skill能做到的**：给你一套可操作的提问框架、准备方法论和叙事技巧，让你的下一次访谈/播客/深度对话的质量显著提升。
- **适用场景**：商业访谈、播客主持、深度对谈、投资人尽调访谈、高管1on1、用户深度访谈
- **不适用场景**：日常闲聊、街头采访、快节奏短视频采访、需要快速获取信息的结构化问卷

## 快速启动清单

如果你明天就要做一场深度访谈，按这个清单准备：

- [ ] 找到访谈发生时的一个"时间锚点"（节日/季节/行业事件），写一个场景式开场
- [ ] 把嘉宾所有公开言论过一遍，标出可以"原话回钩"的3-5个点
- [ ] 如果嘉宾之前来过，找到上次的核心观点，设计"第N次访谈"的叙事
- [ ] 找到嘉宾所在行业最近1-2个关键趋势信号（数据/事件/人物变动）——必须是"当下"的，不是半年前的
- [ ] 逐条过「事实校准纪律」表：人物身份、产品关系、融资状态、时间敏感性、行业共识、论证选择
- [ ] 设计4个板块的问题树（行业→技术/业务→个人→感悟），每板块3-5个问题
- [ ] 确保每个板块有1个"杀手级问题"——对方大概率没被问过的
- [ ] 问题全部改短，删掉所有背景铺垫（除非对方可能不知道的信息）
- [ ] 准备好"如果对方回避怎么办"的备用追问路径
- [ ] 准备1-2个"五个字"式问题——用极简指代触及敏感话题
- [ ] 勾勒一幅嘉宾画像（视觉细节+身份标签+一句话定调）
